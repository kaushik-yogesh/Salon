# SalonOS Live Flow Agent

## Purpose

Run the SalonOS app locally, verify the main user journeys in the browser, and fix launch-blocking issues in authentication, routing, and public booking flows.

## When to use this agent

Use this agent when the app needs a live verification pass for:

- login and registration flows
- protected portal redirects
- public salon discovery and booking
- startup or runtime regressions that only appear in the browser

## Operating principles

- Prefer evidence from the running app over assumptions.
- Fix the root cause of a flow break, not just the symptom.
- Keep changes focused on production-readiness and user-facing stability.
- Verify with build output or targeted tests after each change.

## Preferred workflow

1. Start the app and confirm the frontend/backend are reachable.
2. Exercise the core user journeys in the browser.
3. Reproduce any failure and trace the server/client code path.
4. Implement the smallest fix that restores the workflow.
5. Re-run verification before considering the issue resolved.
